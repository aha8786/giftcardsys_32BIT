
# 🧠 1. 전체 시스템 구조 (아키텍처)

이건 이렇게 간다:

```
[바코드 스캐너]
        ↓
[데스크탑 프로그램 (Python)]
        ↓
[로컬 DB (SQLite)]
        ↓
[알림 모듈 (카카오 or SMS)]
```

👉 핵심: **오프라인에서도 동작 + 빠른 처리 + 단순 구조**

---

# 🗄️ 2. DB 설계 (중요 — 이게 핵심이다)

로컬 DB는 👉 **SQLite 추천 (가볍고 빠름 + 설치 필요 없음)**

## 테이블 구조

### 1) users (회원)

* id (PK)
* phone_number (전화번호)
* created_at

---

### 2) cards (카드)

* id (PK)
* barcode (고유값)
* user_id (FK)
* balance (현재 잔액)
* created_at

---

### 3) transactions (거래 내역)

* id (PK)
* card_id (FK)
* type (충전 / 사용)
* amount
* balance_after
* created_at

---

👉 포인트

* **balance를 cards에 따로 저장 → 속도 최적화**
* 모든 기록은 transactions에 남김 → 감사/추적 가능

---

# ⚙️ 3. 핵심 기능 로직 설계

## 1) 카드 스캔 로직

```
바코드 입력 →
DB 조회 →
IF 카드 존재:
    사용자 정보 + 잔액 + 내역 출력
ELSE:
    신규 등록 화면 이동
```

---

## 2) 카드 등록

```
전화번호 입력 →
초기 충전 금액 입력 →
카드 생성 →
users + cards + transactions 생성
```

---

## 3) 결제 (핵심 로직)

```
입력 금액 > 현재 잔액:
    오류 ("잔액 부족")
ELSE:
    balance 차감
    transactions 기록
```

---

## 4) 충전

```
금액 입력 →
balance 증가 →
transactions 기록
```

---

## 5) 관리자 통계

기간 기준:

* 총 충전 금액
* 총 사용 금액
* 현재 잔액 총합

👉 SQL 핵심:

```
SUM(CASE WHEN type='충전' THEN amount END)
SUM(CASE WHEN type='사용' THEN amount END)
```

## 💻 개발용 프롬프트

```
Python 기반 데스크탑 프로그램을 개발하고 싶다.

목표:
기프트카드 관리 시스템 (충전, 결제, 조회 기능 포함)

요구사항:

1. PySide6를 사용한 GUI 프로그램
2. SQLite 로컬 DB 사용
3. 다음 기능 구현:

[카드 시스템]
- 바코드 기반 카드 식별
- 신규 카드 등록 (전화번호 + 초기 충전금액)
- 기존 카드 조회

[결제 기능]
- 금액 입력 후 잔액 차감
- 잔액 부족 시 오류 처리

[충전 기능]
- 금액 입력 후 잔액 증가

[거래 기록]
- 모든 거래는 transactions 테이블에 저장
- type: "충전" 또는 "사용"

[관리자 기능]
- 기간별 거래 조회
- 총 충전 / 총 사용 / 잔액 합계 표시

[DB 구조]
- users (id, phone_number, created_at)
- cards (id, barcode, user_id, balance, created_at)
- transactions (id, card_id, type, amount, balance_after, created_at)

[예외 처리]
- 잘못된 입력 방지
- DB 연결 실패 처리
- 잔액 부족 처리

[UI]
- 메인 화면 (바코드 입력)
- 카드 등록 화면
- 결제/충전 화면
- 관리자 통계 화면

[추가 요구사항]
- 코드 구조는 모듈화 (db, ui, service 분리)
- 유지보수 가능하게 작성
- 실행 시 DB 자동 생성

이 요구사항을 기반으로
1. 프로젝트 폴더 구조
2. 전체 코드
3. 실행 방법
을 단계적으로 작성해줘.
```
